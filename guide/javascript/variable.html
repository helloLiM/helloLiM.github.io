    # 变量

    javascript的变量是松散类型的，意思是变量可以用于保存任何类型的数据。每个变量只是一个用于保存任意值命名的占位符。有三种方式可以声明变量： var、let和const。

    # 1、var

    var message
    以上就定义了一个message的变量了，你可以当即赋值，也可以之后再赋值

    # 1.1 var 声明作用域

    使用var定义的变量会成为包含它的函数的局部变量：

    function test() {
    var message = 'nihao';
}
test();
console.log(message); //报错

    在全局作用域声明的变量会成为window对象的属性

    var message = 'nihao';
console.log(message); //nihao
console.log(window.message); //nihao

    # 1.2 var 声明提升

    使用var声明变量时，会将你的对这个关键字的声明提升到函数作用域的顶部（但是赋值不会提升）：

    function test1() {
    console.log(message);
    var message = 'nihaoya';
}
test1(); //undefined

    因为变量提升，所以不会有报错，可以看做是

    function test1() {
    var message;
    console.log(message);
    message = 'nihaoya';
}
test1(); //undefined

    # 1.3 var 重复声明

    使用var反复声明同一个变量不会有问题

    var message = 'nihao';
var message = 'woshi';
var message = 'azhu';
console.log(message);  // azhu

    # 2、let

    let 和 var的作用类似，都是用来声明一个变量，但是二者有着很重要区别

    # 2.1 let的声明作用域

    let声明的范围是块级作用域，和var的函数作用域不同：

        {
        var a = 'nihao';
    }
    console.log(a) //nihao
    {
        let b = 'woshi';
    }
    console.log(a) //报错

    # 2.2 let 暂时性死区

    使用let声明变量时，会并不会对你的声明进行提升，所以你在使用let声明变量之前调用这个变量将会报错：

    console.log(a);//undefined
var a = 'nihaoya';

console.log(b);//报错
let b = 'nihaoya';

    这是因为在let声明之前的执行瞬间会形成“暂时性死区”（temporal dead zone）,在这段let声明代码之前引用该变量都会报错。

    # 2.3 let 全局声明

    和var不同，let在全局声明的变量不会成为window对象的属性

    let message = 'nihao';
console.log(message); //nihao
console.log(window.message); //undefined

    # 2.4 let 重复声明

    let 在同一块级作用域内不可重复声明，var虽然可以重复声明，但不可以同let一起对单个变量进行重复声明：

    let message = 'nihao';
let message = 'woshi';//报错

    let message = 'nihao';
var message = 'woshi';//报错

    var message = 'woshi';
let message = 'nihao';//报错

    # 3、const

    const和let的用法基本相同，唯一的区别是const在声明变量时必须赋值，而且不能对const声明的变量指向进行修改：

    let message；
console.log(message);//undefined
const message1;//报错

    const message = 'nihao';
message = 'nihaoya';//报错
const person = {name: 'azhu'};
person.name = lm; //ok
const person1 = {name1: 'azhu'};
person1 = {sex: 'men'}; //报错

    如果const声明的变量是一个对象数据，改变这个对象的内部属性并不违反const限制，只要你不改变这个对象的指向就可以
